B. Create the support route

Create artifacts/api-server/src/routes/support.ts

Requirements

Use Express Router.
Import db from @workspace/db
Import supportRequests from @workspace/db/schema

Create POST /support-requests

Request body shape

{
  name: string,
  email: string,
  phone?: string,
  topic: string,
  preferredContact: "email" | "phone",
  message: string
}

Validation requirements

name required after trim
email required after trim
email must contain "@"
topic required after trim
preferredContact must be "email" or "phone"
message required after trim and be at least 10 characters
if preferredContact is "phone", allow phone to be empty for now but keep the field if provided
normalize blank phone to null

Client identity

Read x_client_id header if present.
Trim it.
If it matches ^client_[a-z0-9_]+$, store it as clientId.
If missing or invalid, store "anonymous_support".
Do not reject the request just because client id is missing.

Insert the record and return 201 with JSON:
{
  id,
  status,
  createdAt
}

Error handling

Return 400 for validation problems with a clear error message.
Return 500 with { error: "Failed to submit support request" } on unexpected failure.
