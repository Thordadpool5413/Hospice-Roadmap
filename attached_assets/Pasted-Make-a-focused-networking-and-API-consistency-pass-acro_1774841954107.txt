A. Create a shared API client utility

Create artifacts/mobile/services/apiClient.ts

This file should become the single source of truth for mobile API requests.

Required exports

1. apiBase(): string

Behavior:
Use the same environment aware resolution that currently allows Expo and Replit local development to work.
If similar logic already exists in aiService.ts or cmsProviderService.ts, move that logic here and reuse it.
Keep behavior compatible with the current dev environment.
Do not hardcode a production only host.

2. makeRequestTimeoutSignal(timeoutMs?: number): AbortSignal

Behavior:
Create a timeout based AbortController signal similar to the current makeSignal patterns.
Default timeout should be one consistent value, such as 12000 ms, unless specific calls need longer.
If cleanup helper logic is useful, include it.
Keep it simple.

3. mergeJsonHeaders(extraHeaders?: Record<string, string>): Record<string, string>

Behavior:
Return a clean headers object with:
Accept: application/json
Content Type: application/json
plus any extra headers merged in.
If a request needs non JSON behavior later, that can bypass this helper.

4. parseJsonResponse<T>(res: Response): Promise<T>

Behavior:
If res.ok is true:
attempt to parse JSON and return it
if parsing fails, throw Error("Invalid server response")

If res.ok is false:
attempt to parse JSON and read:
error
message
or a similar field
throw Error with the most useful message available
fallback to:
"Request failed"

5. fetchJson<T>(url: string, options?: RequestInit & { timeoutMs?: number }): Promise<T>

Behavior:
Use makeRequestTimeoutSignal
merge JSON headers if the caller did not explicitly provide headers
call fetch
return parseJsonResponse<T>(res)

Allow callers to pass their own headers and method.
Preserve custom headers such as x_client_id.

E. Create one shared app network hook

Create artifacts/mobile/hooks/useAppNetwork.ts

Requirements

1. Use the same underlying network information source the app already uses.
2. If the project already uses NetInfo, reuse it.
3. Export:
useAppNetwork()

Return shape:
{
  isOnline: boolean;
  isInternetReachable: boolean | null;
  hasCheckedNetwork: boolean;
}

Behavior:
Treat null unknown states carefully.
Recommended logic:
isOnline should only be false when network is clearly unavailable.
If the state is unknown, default isOnline to true until proven offline, to avoid false negatives on launch.
hasCheckedNetwork should become true after the first network state event.

Keep the hook simple and reusable.
Use cleanup correctly.
